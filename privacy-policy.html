Privacy Policy

This page describes how the current FORGE CUSTOMS demo site handles information. The site is frontend-only and does not submit checkout or inquiry data to a backend service.

What This Demo Stores

The current site stores limited interaction data in your browser so the experience works like a storefront.

Cart Data

Selected systems and custom PC configurations are stored in browser local storage so the cart stays available between page reloads.

Inquiry State

Build inquiry forms show a frontend confirmation state, but the current demo does not send form contents to a server.

Interface Preferences

Normal browser behavior may cache page assets and keep form fields available based on your device and browser settings.

What This Demo Does Not Do

These limits matter because this project is currently a static frontend experience.

  • No payment processing occurs through this demo site.
  • No account creation or login system is present.
  • No order database or CRM submission is connected to the inquiry form.
  • No backend customer support portal is collecting tickets from this page.

Third-Party Requests

The site uses a few third-party services needed for rendering and outbound support links.

Google Fonts

Pages load web fonts from Google Fonts. Visiting the site may trigger standard font asset requests to Google.

Hosting Provider Logs

If the site is deployed on a platform such as GitHub Pages, the hosting provider may maintain standard request logs outside this frontend codebase.

External Support Links

Driver download and contact links may take you to third-party sites that apply their own privacy policies.

Your Choices

You can control most data used by this demo directly from your browser.

  • Clear browser local storage to remove saved cart items and custom build selections.
  • Close the page without submitting anything to avoid reusing inquiry form text.
  • Disable third-party content at the browser level if you do not want remote font requests.

If FORGE CUSTOMS later adds live forms, analytics, checkout, or account features, this policy should be expanded before those services go live.
